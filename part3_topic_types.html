Part III: Topic Types

This module introduces core DITA topic types and their appropriate usage to improve content clarity and organization.

Learning Objectives

Upon completing this module, you will be able to:

  • Explain the purpose and appropriate use cases for each of the core DITA topic types.
  • Identify the common structural elements (tags) associated with concept, task, reference, glossary, and troubleshooting topics.
  • Apply best practices for structuring content within each topic type to improve clarity and usability.
  • Understand how different topic types work together to form comprehensive documentation.

DITA defines several structured topic types, each intended for a different documentation purpose. Using the correct topic type ensures content is focused, reusable, and well-organized across deliverables. Topic types are not arbitrary categories; they reflect how users actually consume technical information. By matching content structure to user intent, DITA topics become more findable, scannable, and actionable.

  • Concept Topics
    Concept topics introduce and explain subjects, providing background knowledge before procedural or reference content.
  • Task Topics
    Task topics provide clear, step-by-step instructions for completing specific procedures and reducing user errors.
  • Reference Topics
    Reference topics provide structured, factual data used to support task and concept topics.
  • Glossary Entries
    Glossary topics define specialized terms and ensure consistent usage across documentation and translations.
  • Troubleshooting Topics
    Troubleshooting topics guide users through problem identification and resolution, improving self-service and reducing support costs.
